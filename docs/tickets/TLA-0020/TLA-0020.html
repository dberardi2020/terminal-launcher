TLA-0020 — A family of management commands

Installing, updating, removing and repairing Terminal Launcher is currently a set of per-platform scripts a user has to know about. There's no single, discoverable way to ask the tool to manage itself.

Current state — what a user has to know

TaskToday
Install the CLIpipx install git+…, or clone and python -m terminal_launcher
Install the Dock app (macOS)./packaging/install-macos.sh
Install the Start Menu entry (Windows)packaging\windows\install-shortcut.ps1
Install /restoreintegrations/claude-code/install.sh or install.ps1
Update anythingre-run whichever of the above applies
Uninstalldelete things by hand
Repairno path at all

Four installers, two of them platform-paired, and no uninstall or update story.

The failure mode that motivates it

The /restore installers bake absolute paths (interpreter + entry point) into ~/.claude/commands/restore.md at install time. Move or rename the checkout and the command silently breaks — it points at something that no longer exists, and nothing tells the user why. A repair verb that re-resolves and rewrites those paths is the concrete case; "my install got weird" is the general one.

Related known repair cases:

  • Missing or rebuilt venv (the interpreter path goes stale the same way).
  • An unsigned .app re-prompting for Automation permission after each rebuild (TLA-0003).
  • A stale Dock icon after an in-place reinstall (macOS caches by path).

Candidate surface

update · install · uninstall · reinstall · repair — as Claude Code skills, CLI verbs, or both. Worth deciding which: a CLI verb works everywhere and is scriptable; a skill is more discoverable from inside a pane, which is where users already are.

Overlap — decide before building

This is not a fresh idea, and three tickets now circle the same ground:

  • TLA-0012 — design the install/setup and maintenance/update flows for going live. The umbrella.
  • TLA-0010 — streamline pushing updates to the installed Dock app (py2app alias mode, symlinked package, or running the GUI from source).
  • TLA-0020 (this) — the user-facing command surface over all of it.

First action on this ticket is a merge decision, not code: does TLA-0020 supersede TLA-0012 and TLA-0010, or is it the interface layer on top of them? Building the commands before settling that risks a third partial answer to the same question.
