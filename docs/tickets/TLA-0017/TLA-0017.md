# TLA-0017 — GUI automation / demo hooks

There is no supported way to open the visual composer **in a given state**. It always
starts with an empty composer, and loading a workspace into it is a mouse click.

## Why now

The README demo screenshots needed a populated composer — a loaded quad, the slot editor
open, the pane manager open. Getting them required a throwaway harness: launch pywebview
against a fake config, wait for `pywebviewready` → `refresh()` to settle, then reach into
the page with `window.evaluate_js("loadWorkspace(0); startEditing(0)")`.

That worked, but it is not repeatable by anyone else, it depends on internal JS function
names (`loadWorkspace`, `startEditing`, `#panesBtn`), and it silently rots the moment
`builder.html` is refactored. Regenerating the screenshots after a UI change currently
means rebuilding that harness from scratch.

## Proposed surface

**Minimum — and useful on its own:**

```sh
terminal-launcher gui --load "Full Stack"
```

Opens the composer with that workspace already loaded. This is not just a test hook: it's
a genuinely nice everyday path (`edit this workspace visually`), which is the main argument
for building it rather than keeping a private harness.

**Optional extras, for screenshots and UI tests:**

- `--edit-slot <n>` — open the inline slot editor on a given slot.
- `--panel panes` — open the pane-library side panel.
- A screenshot mode that loads a supplied config, reaches a state, captures the window,
  and exits — so `docs/assets/*.png` can be regenerated by one command.

## Design notes

- Drive it **through the Python API**, not by poking JS function names. The state already
  exists server-side (`Api.get_state`); the flags should set the composer's initial state
  before the page renders, rather than mutating it afterwards.
- A fake/demo config is a dependency here — see
  [TLA-0016](../TLA-0016/TLA-0016.md) for building one non-interactively.

## Value

Lower priority than TLA-0016 because its main beneficiary is demo/test tooling, not
end users — but it retires an unmaintainable harness and makes the README's screenshots
reproducible instead of hand-made.
